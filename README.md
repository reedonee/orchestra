# Orchestra - React Native Windows (WinUI 3)

A React Native application targeted exclusively for Windows using **react-native-windows** with C++ and WinUI 3 (Windows App SDK).

## Project Structure

```
Orchestra/
├── src/                      # Source files (components, screens, utilities)
├── windows/                  # Windows-specific code (generated by react-native-windows-init)
├── App.tsx                   # Root app component
├── index.js                  # Entry point
├── app.json                  # App configuration
├── package.json              # Dependencies & scripts
├── metro.config.js           # Metro bundler configuration (with .cjs support)
├── babel.config.js           # Babel configuration
├── jest.config.js            # Jest testing configuration
├── tsconfig.json             # TypeScript configuration
├── .eslintrc.js              # ESLint configuration
└── react.config.js           # React Native CLI configuration
```

## Prerequisites

Before setting up Orchestra, ensure you have:

1. **Node.js** (LTS) and **npm** (or yarn)
2. **Visual Studio 2022** (Community, Professional, or Enterprise) with:
   - "Desktop development with C++" workload
   - "Universal Windows Platform development" workload
   - Windows 10/11 SDK installed
3. **Git** available in your PATH

## Setup Instructions

### 1. Install Dependencies

```powershell
cd Orchestra
npm install
```

### 2. Add React Native Windows Support

Run the React Native Windows initialization command to generate the Windows project structure with C++ and WinUI 3:

```powershell
npx react-native-windows-init --overwrite --language cpp --useWinUI3
```

This command will:
- Create the `windows/` directory with Visual Studio solution and C++ project files
- Generate WinUI 3 / Windows App SDK integration
- Install `react-native-windows` as a dev dependency (if not already present)

### 3. Verify metro.config.js

The `metro.config.js` is already configured to support `.cjs` file resolution. No additional changes needed.

### 4. Build & Run

#### Option A: Using React Native CLI (Recommended)

```powershell
npm run windows
```

This command will:
- Start the Metro bundler
- Build the Windows app (Debug, x64 by default)
- Launch the app

#### Option B: Using Visual Studio

1. Open `windows/Orchestra.sln` in Visual Studio 2022
2. Set the startup project to **Orchestra** (if not already set)
3. Select the appropriate build configuration:
   - **Debug** or **Release**
   - **x64** (or **ARM64** / **x86** if needed)
4. Press **F5** or click **Run** to build and launch

## Development Workflow

### Hot Reload
While the Metro bundler is running and the app is open:
- Press **Ctrl+M** (or **Cmd+M** on Mac) in the app to open the developer menu
- Select "Reload" to hot-reload your changes
- Select "Enable Fast Refresh" for automatic reloading on file save

### Debugging
1. Open the Developer Menu (Ctrl+M) in the running app
2. Select "Open Debugger" to start debugging with Chrome DevTools
3. Or attach the Visual Studio debugger directly to the running process

### Metro Bundler Troubleshooting
If Metro doesn't start automatically:
```powershell
npm run start
```
Then in another terminal, run:
```powershell
npm run windows
```

## Troubleshooting

### Visual Studio / MSBuild Not Found
- Ensure Visual Studio 2022 is installed with required workloads
- Alternatively, open "Developer Command Prompt for VS 2022" before running npm commands
- Or use Visual Studio's solution file directly

### Metro Fails to Resolve Modules
- Verify `metro.config.js` includes `.cjs` in `sourceExts`
- Restart the Metro bundler: Stop and run `npm run start` again

### Windows SDK or WinAppSDK Not Found
- Open `windows/Orchestra.sln` in Visual Studio
- Visual Studio will notify you of missing dependencies
- Install required packages via Visual Studio Installer or NuGet

### Build Fails with MSVC Errors
- Ensure "Desktop development with C++" workload is installed
- Check that MSVC toolset is up to date in Visual Studio Installer
- Try cleaning the build: Delete `windows/build` folder and rebuild
