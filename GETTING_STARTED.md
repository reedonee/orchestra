# Orchestra - Quick Start Guide

## Prerequisites Verification

Before running the setup, ensure you have:

### 1. Node.js & npm
```powershell
node --version   # Should be v16 or higher
npm --version    # Should be v7 or higher
```

If not installed:
- Download from [nodejs.org](https://nodejs.org/)
- Choose the **LTS (Long Term Support)** version
- Run the installer and follow prompts

### 2. Visual Studio 2022
Install **Visual Studio 2022** (Community Edition is free):
- Download from [visualstudio.microsoft.com](https://visualstudio.microsoft.com/)
- Choose **Community Edition**
- During installation, select these workloads:
  - ✅ **Desktop development with C++**
  - ✅ **Universal Windows Platform development**
  - ✅ Optional: **.NET desktop development** (if building .NET interop)

### 3. Windows SDK & Windows App SDK
- Install through Visual Studio Installer
- Required: Windows 10/11 SDK (version 1909 or later recommended)
- Required: Windows App SDK for WinUI 3 support

### 4. Git (Optional but Recommended)
```powershell
git --version
```
Download from [git-scm.com](https://git-scm.com/) if needed.

```powershell

# Run the setup script
.\setup.bat
```

This script will:
1. ✅ Verify Node.js and npm are installed
2. ✅ Install all npm dependencies
3. ✅ Initialize React Native Windows with C++ and WinUI 3
4. ✅ Display next steps

```bash
cd ~/path/to/Orchestra

```

## Manual Setup (Step-by-Step)

If you prefer to run commands individually:

### Step 1: Install Dependencies
```powershell
cd c:\Users\hp\Desktop\red\create\Orchestra
npm install
```

### Step 2: Add React Native Windows
```powershell
npx react-native-windows-init --overwrite --language cpp --useWinUI3
```

**What this does:**
- Creates `windows/` directory with Visual Studio solution
- Generates C++ WinRT project files
- Configures WinUI 3 / Windows App SDK integration
- Installs `react-native-windows` package

### Step 3: Build & Run

#### Option A: Using npm (Easiest)
```powershell
npm run windows
```

This will:
- Start the Metro bundler
- Build the Windows app
- Launch the app window

#### Option B: Using Visual Studio
1. Navigate to `windows/Orchestra.sln`
2. Right-click → **Open with** → **Visual Studio 2022**
3. Select **Orchestra** as the startup project
4. Choose **Release** or **Debug** configuration
5. Choose **x64** (or **ARM64** for ARM devices)
6. Press **F5** or click **Run** button

## Troubleshooting

### ❌ "npm: command not found"
- **Solution**: Node.js/npm not installed or not in PATH
- Install from [nodejs.org](https://nodejs.org/)
- Restart terminal/PowerShell after installation

### ❌ "Cannot find Visual Studio / MSBuild"
- **Solution**: Visual Studio 2022 not installed or not in PATH
- Install Visual Studio 2022 with required workloads
- Or open "Developer Command Prompt for VS 2022" and run commands there

### ❌ "Windows SDK not found"
- **Solution**: Windows SDK not installed
- Open Visual Studio Installer → Modify → Check "Windows SDK" box
- Install the latest Windows SDK (v22H2 recommended)

### ❌ "Metro bundler fails to start"
- **Solution**: Port 8081 already in use or Metro cache corrupted
- Run: `npm run start` in a separate terminal
- Or kill processes on port 8081: `netstat -ano | findstr :8081`

### ❌ "Build fails with C++ compilation errors"
- **Solution**: MSVC compiler or Windows App SDK missing
- Ensure "Desktop development with C++" workload is installed in VS
- Update Visual Studio to latest version
- Delete `windows/build` folder and rebuild

### ❌ "WinUI 3 packages not found"
- **Solution**: Windows App SDK NuGet packages not installed
- Open `windows/Orchestra.sln` in Visual Studio
- Visual Studio will prompt to install missing packages
- Accept and install

## Development Workflow

### Starting Development

```powershell
# Terminal 1: Start Metro bundler
npm run start

# Terminal 2: Build and run Windows app
npm run windows
```

### Editing Code

1. Edit components in `src/` or modify `App.tsx`
2. Save the file
3. App should hot-reload automatically (if Fast Refresh is enabled)
4. If not: Press **Ctrl+M** in the app → **Reload**

### Building for Release

```powershell
npm run build-windows
```
*(Or use Visual Studio with Release configuration)*

## Folder Structure

```
Orchestra/
├── src/                          # Your custom components and screens
├── windows/                      # ⚠️ Generated by react-native-windows-init
│   ├── Orchestra.sln             # Visual Studio solution
│   ├── Orchestra/                # C++ / WinUI 3 app project
│   └── packages.config           # NuGet packages
├── __tests__/                    # Jest test files
├── App.tsx                       # Root React component
├── index.js                      # App entry point
├── app.json                      # App configuration (name, version, etc.)
├── package.json                  # Dependencies and scripts
├── metro.config.js               # Metro bundler config (handles .cjs files)
├── babel.config.js               # Babel transpiler config
├── jest.config.js                # Jest test runner config
├── tsconfig.json                 # TypeScript compiler config
├── .eslintrc.js                  # ESLint code quality config
├── .prettierrc.json              # Code formatter config
├── .gitignore                    # Git ignore patterns
├── setup.bat                     # Setup script for Windows
├── setup.sh                      # Setup script for macOS/Linux
├── README.md                     # Full documentation
└── GETTING_STARTED.md            # This file
```
